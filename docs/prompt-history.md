## Lessons Learned from GenAI Usage

### What AI does well
- Project scaffolding — generates files, projects, and configuration quickly
- Repetitive boilerplate — CRUD repositories, DTOs, mappings
- Test structure — test class setup and obvious test cases
- Docker — well-formatted Dockerfiles and docker-compose

### What always requires manual review
- **Security**: AI tends toward insecure defaults — plain-text passwords, `SELECT *`, queries without parameters
- **Project constraints**: AI defaults to EF, Dapper, MediatR because they are the common patterns — constraints must be stated explicitly in the prompt and re-verified in the output
- **UTC vs. local time**: AI frequently uses `DateTime.Now` instead of `DateTime.UtcNow`
- **Data isolation**: Queries without a user filter are generated frequently and are a critical security issue
- **NuGet packages**: AI may forget to add required packages or generate version conflicts
- **Angular decorators**: `@Input()` and `@Output()` are frequently omitted in generated components

### Validation strategy adopted
1. Build the solution — verify it compiles with no errors
2. Check project dependency graph — no layer violations
3. Search for `SELECT *`, `DateTime.Now`, and string interpolation in SQL
4. Verify passwords are never stored as plain text
5. Run the full test suite — all tests must pass
6. Execute `docker-compose up --build` end-to-end and verify functionality
